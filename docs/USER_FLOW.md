# User Flow

## 1. 개요

칸반보드 사용자의 주요 흐름을 정의한다. v2.0부터 인증이 필수이며, 로그인 후 보드가 표시된다.

---

## 2. 전체 흐름도

```
[브라우저에서 URL 접속]
         │
         ▼
  [세션 확인 (Supabase)]
         │
    ┌────┴──────┐
    │           │
    ▼           ▼
[미로그인]   [로그인 상태]
    │           │
    ▼           ▼
[인증 패널]  [보드 표시]
    │
  ┌─┴──────────────┐
  │                │
  ▼                ▼
[이메일 로그인]  [GitHub 로그인]
  │                │
  ▼                └──► [GitHub 인증 → 리다이렉트]
[로그인 성공]              │
  │            ┌──────────┘
  └────────────┘
         │
         ▼
    [보드 표시]
  ┌───────────────────────────────┐
  │  Todo  │  In Progress  │ Done │
  │ [카드] │    [카드]     │[카드]│
  └───────────────────────────────┘
         │
   ┌─────┴──────────────┐──────────────┐
   │                    │              │
   ▼                    ▼              ▼
[카드 추가]          [카드 드래그]   [로그아웃]
```

---

## 3. 시나리오별 상세 흐름

### Flow 1: 앱 진입 (미로그인)

```
사용자 → URL 접속
       → Supabase 세션 확인 (INITIAL_SESSION)
       → 세션 없음 → 인증 패널 표시 (로그인 뷰)
```

### Flow 2: 앱 진입 (로그인 상태)

```
사용자 → URL 접속
       → Supabase 세션 확인
       → 유효한 세션 있음 → 보드 표시
       → DB에서 카드 목록 로드
       → 카드가 없으면 샘플 카드 자동 삽입
```

---

### Flow 3: 이메일 회원가입

```
사용자 → [회원가입] 링크 클릭 → 회원가입 뷰 표시
       → 이메일 + 비밀번호 입력 → [회원가입] 버튼 클릭
       → Supabase signUp 호출
       → 이메일 인증 안내 뷰 표시
       → 이메일함에서 인증 링크 클릭
       → 앱 URL로 리다이렉트 (GitHub Pages)
       → 자동 로그인 → 보드 표시
```

### Flow 4: 이메일 로그인

```
사용자 → 이메일 + 비밀번호 입력 → [로그인] 버튼 클릭
       → Supabase signInWithPassword 호출

  [인증 실패]
       → 에러 메시지 표시

  [인증 성공]
       → onAuthStateChange(SIGNED_IN) 이벤트
       → 보드 표시 + DB 카드 로드
```

### Flow 5: GitHub OAuth 로그인

```
사용자 → [GitHub로 로그인] 버튼 클릭
       → Supabase signInWithOAuth({ provider: 'github' }) 호출
       → GitHub 인증 페이지로 리다이렉트
       → GitHub에서 앱 권한 승인
       → Supabase 콜백 URL로 리다이렉트
       → 앱 URL(GitHub Pages)로 리다이렉트
       → onAuthStateChange(SIGNED_IN) 이벤트
       → 보드 표시 + DB 카드 로드
```

---

### Flow 6: 카드 추가

```
사용자 → 원하는 컬럼 하단 입력 필드 클릭
       → 텍스트 입력
       → [추가] 버튼 클릭 또는 Enter 키 입력

  [빈 텍스트인 경우]
       → 카드 생성 안 됨

  [텍스트 있는 경우]
       → Supabase insert 호출 (user_id, column_id, text)
       → 반환된 카드 데이터로 DOM에 카드 추가
       → 입력 필드 초기화
       → 컬럼 카드 수 뱃지 갱신
```

### Flow 7: 카드 드래그앤드롭 이동

```
사용자 → 카드 위에 마우스 버튼 누름 (dragstart)
       → 카드 투명도 감소 (시각 피드백)

       → 카드를 다른 컬럼 위로 드래그
       → 드롭 가능 컬럼 배경색 변경 + 점선 테두리 (drag-over 피드백)

  [다른 컬럼에 드롭]
       → 기존 컬럼에서 카드 제거
       → 대상 컬럼에 카드 추가
       → Supabase update({ column_id }) 호출
       → 양쪽 컬럼 카드 수 뱃지 갱신
       → 강조 효과 해제 (dragend)
```

### Flow 8: 카드 삭제

```
사용자 → 삭제할 카드의 × 버튼 클릭
       → Supabase delete().eq('id', cardId) 호출
       → 카드 DOM 제거
       → 해당 컬럼 카드 수 뱃지 갱신
```

### Flow 9: 로그아웃

```
사용자 → 헤더의 [로그아웃] 버튼 클릭
       → Supabase signOut() 호출
       → onAuthStateChange(SIGNED_OUT) 이벤트
       → 보드 숨김 → 인증 패널 표시 (로그인 뷰)
```

---

## 4. 엣지 케이스

| 케이스 | 처리 방법 |
|--------|---------|
| 빈 텍스트로 카드 추가 시도 | 카드 생성 차단 |
| 드래그 중 Escape 키 | 드래그 취소, 카드 원위치, 강조 해제 |
| 모든 카드 삭제 후 보드 | 빈 컬럼 정상 표시, 추가 폼 유지 |
| 이메일 인증 전 로그인 시도 | Supabase에서 에러 반환, 에러 메시지 표시 |
| 잘못된 비밀번호 | Supabase에서 에러 반환, 에러 메시지 표시 |
| DB 요청 실패 | 콘솔 에러 (v3.0에서 UI 에러 처리 예정) |

## 5. 화면 상태 요약

```
[인증 패널]
  ├─ view-signin  (기본)
  ├─ view-signup
  └─ view-verify

[보드]
  └─ board-wrapper (로그인 후)
       ├─ header (이메일 + 로그아웃)
       └─ .board
            ├─ #todo
            ├─ #in-progress
            └─ #done
```

상태 전환은 DOM의 `hidden` 클래스 토글로만 처리한다. 페이지 이동 없음.

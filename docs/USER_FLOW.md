# User Flow

## 1. 개요

칸반보드 사용자의 주요 흐름을 정의한다.

---

## 2. 전체 흐름도

```
[브라우저에서 index.html 열기]
          │
          ▼
[보드 렌더링]
  ┌───────────────────────────────┐
  │  Todo  │  In Progress  │ Done │
  │ [카드] │    [카드]     │[카드]│
  └───────────────────────────────┘
          │
    ┌─────┴──────────────┐
    │                    │
    ▼                    ▼
[카드 추가]          [카드 드래그]
    │                    │
    ▼                    ▼
[텍스트 입력]      [다른 컬럼에 드롭]
    │                    │
    ▼                    ▼
[추가 버튼/Enter]   [컬럼 이동 완료]
    │
    ▼
[카드 생성 & 표시]
    │
    ▼
[카드 삭제 버튼(×) 클릭]
    │
    ▼
[카드 제거]
```

---

## 3. 시나리오별 상세 흐름

### Flow 1: 앱 진입

```
사용자 → 브라우저에서 index.html 열기
       → 보드 레이아웃 표시
       → 3개 컬럼(Todo / In Progress / Done) 렌더링
       → 각 컬럼에 샘플 카드 2~3개 표시
       → 입력 폼 활성화 대기
```

---

### Flow 2: 카드 추가

```
사용자 → 원하는 컬럼 하단 입력 필드 클릭
       → 텍스트 입력
       → [추가] 버튼 클릭 또는 Enter 키 입력

  [빈 텍스트인 경우]
       → 카드 생성 안 됨 (입력 필드 포커스 유지)

  [텍스트 있는 경우]
       → 해당 컬럼에 새 카드 추가
       → 입력 필드 초기화
       → 컬럼 카드 수 뱃지 갱신
```

---

### Flow 3: 카드 드래그앤드롭 이동

```
사용자 → 카드 위에 마우스 버튼 누름 (dragstart)
       → 카드 투명도 감소 (시각 피드백)

       → 카드를 다른 컬럼 위로 드래그
       → 드롭 가능 컬럼 배경색 변경 + 점선 테두리 (drag-over 피드백)

  [같은 컬럼에 드롭]
       → 컬럼 내 위치 이동 없음 (현재 v1.0 동작)

  [다른 컬럼에 드롭]
       → 기존 컬럼에서 카드 제거
       → 대상 컬럼에 카드 추가
       → 양쪽 컬럼 카드 수 뱃지 갱신
       → 강조 효과 해제 (dragend)
```

---

### Flow 4: 카드 삭제

```
사용자 → 삭제할 카드의 × 버튼 클릭
       → 카드 즉시 제거
       → 해당 컬럼 카드 수 뱃지 갱신
```

---

## 4. 엣지 케이스

| 케이스 | 처리 방법 |
|--------|---------|
| 빈 텍스트로 카드 추가 시도 | 카드 생성 차단, 입력 필드 포커스 유지 |
| 드래그 중 Escape 키 | 드래그 취소, 카드 원위치, 강조 해제 |
| 모든 카드 삭제 후 보드 | 빈 컬럼 정상 표시, 추가 폼 유지 |
| 빠른 연속 드래그 | 각 드롭 이벤트 독립 처리 |

---

## 5. 화면 전환 요약

```
[단일 페이지 앱 — 화면 전환 없음]

index.html (항상 동일)
  └─ 상태 변화는 DOM 조작으로만 처리
  └─ 새로고침 시 초기 샘플 카드 상태로 리셋
```

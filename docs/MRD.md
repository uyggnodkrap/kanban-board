## 2. 시장 기회

개인 생산성 도구 시장에서 "가볍고 빠른 개인 칸반보드" 니즈가 존재한다.

- 기존 SaaS 툴(Trello, Linear, Jira)은 기능이 많아 개인 용도로 무겁다.
- Supabase를 활용하면 백엔드 없이도 멀티 디바이스 동기화가 가능하다.
- 순수 HTML/CSS/JS + Supabase 조합으로 빌드 툴 없이 즉시 배포할 수 있다.

## 3. 타겟 사용자

### 주요 타겟

| 세그먼트 | 특징 | 핵심 니즈 |
|---------|------|---------|
| 개인 개발자 | 사이드 프로젝트 진행 중 | 어디서든 접근 가능한 태스크 추적 |
| 학습자/수강생 | 코딩 부트캠프, 자기계발 | 오늘 할 일 시각화, 데이터 유지 |
| 프론트엔드 입문자 | HTML/CSS/JS + Supabase 공부 중 | 실습 코드 참고 |

### 부차 타겟

- 소규모 팀 (2~3인): 개인 보드로 각자 활용
- Supabase 학습자: 인증·DB 연동 패턴 참고

## 4. 경쟁 분석

| 제품 | 강점 | 약점 | 우리의 차별점 |
|------|------|------|------------|
| Trello | 풍부한 기능, 협업 | 복잡, 유료 제한 | 단순·경량, 무료 |
| Notion | 올인원 워크스페이스 | 무겁고 학습 곡선 높음 | 즉시 사용, 단순함 |
| GitHub Projects | 개발자 친화 | GitHub 의존성 높음 | 독립적 운영 |
| TodoMVC | 단순함 | 영속성 없음 | Supabase DB 동기화 |

## 2. 시장 기회

개인 생산성 도구 시장에서 "즉시 사용 가능한 경량 칸반보드" 니즈가 존재한다.

- 기존 SaaS 툴(Trello, Linear, Jira)은 회원가입·인터넷 연결이 필수다.
- 오프라인 또는 로컬 환경에서 설치 없이 바로 사용 가능한 도구가 부재하다.
- 학습·교육 목적으로 HTML/CSS/JS 기초를 익히는 데 적합한 실습 프로젝트이기도 하다.

---

## 3. 타겟 사용자

### 주요 타겟

| 세그먼트 | 특징 | 핵심 니즈 |
|---------|------|---------|
| 개인 개발자 | 사이드 프로젝트 진행 중 | 가벼운 태스크 추적 |
| 학습자/수강생 | 코딩 부트캠프, 자기계발 | 오늘 할 일 시각화 |
| 프론트엔드 입문자 | HTML/CSS/JS 공부 중 | 실습 코드 참고 |

### 부차 타겟

- 소규모 팀 (2~3인): 공유 없이 개인 보드로 활용
- 오프라인 환경 사용자: 인터넷 없이도 파일만으로 실행

## 4. 경쟁 분석

| 제품 | 강점 | 약점 | 우리의 차별점 |
|------|------|------|------------|
| Trello | 풍부한 기능, 협업 | 인터넷/회원가입 필수 | 오프라인·설치 불필요 |
| Notion | 올인원 워크스페이스 | 무겁고 학습 곡선 높음 | 즉시 사용, 단순함 |
| GitHub Projects | 개발자 친화 | GitHub 계정 필수 | 의존성 없음 |
| 포스트잇 | 직관적, 오프라인 | 디지털 관리 불가 | 디지털 + 간편함 |

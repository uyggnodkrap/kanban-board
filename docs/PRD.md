### 목적

업무 또는 학습 태스크를 시각적으로 관리할 수 있는 경량 칸반보드를 제공한다. 별도 서버나 설치 없이 브라우저만으로 동작한다.

---

## 2. 배경 및 문제 정의

- 할 일을 텍스트 목록으로만 관리하면 진행 상태 파악이 어렵다.
- Trello, Notion 같은 툴은 로그인·계정 없이 즉시 사용하기 불편하다.
- 순수 HTML/CSS/JS로 인터넷 없이도 동작하는 오프라인 칸반보드가 필요하다.

## 4. 기능 요구사항 (Functional Requirements)

### FR-01: 보드 표시
- 3개 컬럼(Todo, In Progress, Done)이 가로로 나열된다.
- 각 컬럼에는 헤더와 카드 목록, 카드 추가 폼이 있다.

### FR-02: 카드 생성
- 각 컬럼 하단 입력 폼에서 텍스트를 입력하고 추가 버튼(또는 Enter)을 누르면 카드가 생성된다.
- 빈 텍스트로는 카드를 생성할 수 없다.

### FR-03: 카드 이동 (드래그앤드롭)
- 카드를 드래그하여 다른 컬럼에 드롭하면 해당 컬럼으로 이동된다.
- 드래그 중인 카드는 시각적으로 구분된다(투명도 감소).
- 드롭 가능한 컬럼은 시각적으로 강조된다.

### FR-04: 카드 삭제
- 카드 우측의 × 버튼을 클릭하면 카드가 삭제된다.

### FR-05: 초기 샘플 데이터
- 앱 최초 로드 시 각 컬럼에 예시 카드가 2~3개 표시된다.

---

## 5. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- 페이지 로드 시간 1초 이내 (로컬 파일 기준)
- 드래그앤드롭 응답 지연 없음

### NFR-02: 호환성
- 최신 Chrome, Firefox, Edge, Safari 지원
- 모바일 터치 이벤트는 v1.0 범위 외 (v2.0 로드맵)

### NFR-03: 접근성
- 컬럼·카드에 적절한 aria 속성 부여 (향후 개선 대상)

### NFR-04: 유지보수성
- HTML/CSS/JS를 각각 별도 파일로 분리하여 단일 책임 원칙 준수

---

## 6. 범위 외 (Out of Scope) — v1.0

- 사용자 인증/로그인
- 데이터 영속성 (새로고침 시 초기화)

- 모바일 터치 드래그앤드롭

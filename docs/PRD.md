### 목적

업무 또는 학습 태스크를 팀원과 함께 실시간으로 관리할 수 있는 칸반보드를 제공한다. Supabase 인증·DB·Realtime을 통해 멀티 유저 환경에서 데이터가 동기화된다.

---

## 2. 배경 및 문제 정의

- 할 일을 텍스트 목록으로만 관리하면 진행 상태 파악이 어렵다.
- 기기를 바꿔도 데이터가 유지되는 칸반보드가 필요하다.
- 팀원과 같은 보드를 공유하고 실시간으로 작업 현황을 파악하고 싶다.
- 빌드 툴 없이 순수 HTML/CSS/JS + Supabase로 구현해 유지보수를 단순화한다.

## 4. 기능 요구사항 (Functional Requirements)

### FR-01: 인증
- 이메일/비밀번호로 회원가입·로그인이 가능하다.
- GitHub OAuth로 소셜 로그인이 가능하다.
- 회원가입 시 이메일 인증 메일이 발송된다.
- 로그인 상태는 세션으로 유지된다.
- 로그아웃 버튼으로 세션을 종료할 수 있다.

### FR-02: 보드 표시
- 로그인 후 3개 컬럼(Todo, In Progress, Done)이 가로로 나열된다.
- 헤더에 현재 로그인한 이메일, 팀원 초대 버튼, 로그아웃 버튼이 표시된다.
- 보드 우측에 활동 로그 사이드 패널이 항상 표시된다.

### FR-03: 카드 생성
- 각 컬럼 하단 입력 폼에서 텍스트를 입력하고 추가 버튼(또는 Enter)을 누르면 카드가 생성된다.
- 생성된 카드는 Supabase DB에 저장된다.
- 빈 텍스트로는 카드를 생성할 수 없다.

### FR-04: 카드 이동 (드래그앤드롭)
- 카드를 드래그하여 다른 컬럼에 드롭하면 해당 컬럼으로 이동된다.
- 이동된 컬럼 정보는 DB에 즉시 반영된다.
- 드래그 중인 카드는 시각적으로 구분된다(투명도 감소).
- 드롭 가능한 컬럼은 시각적으로 강조된다.

### FR-05: 카드 삭제
- 카드 우측의 × 버튼을 클릭하면 카드가 삭제된다.
- 삭제는 DB에 즉시 반영된다.

### FR-06: 데이터 영속성
- 로그인한 사용자의 카드는 새로고침 후에도 유지된다.
- 다른 기기에서 로그인해도 동일한 카드 목록이 표시된다.

### FR-07: 초기 샘플 데이터
- 최초 로그인(카드가 없는 상태) 시 각 컬럼에 예시 카드가 자동 삽입된다.

### FR-08: 보드 공유 (팀원 초대)
- 보드 오너는 "팀원 초대" 버튼을 클릭해 7일 유효 초대 링크를 생성할 수 있다.
- 초대 링크는 클립보드에 자동 복사된다.
- 초대 링크를 받은 사용자는 로그인 후 해당 보드에 자동 합류된다.
- 이미 합류한 멤버가 링크를 다시 열어도 중복 등록되지 않는다.
- 만료된 초대 링크 접근 시 에러 토스트가 표시된다.

### FR-09: 활동 로그
- 카드 추가/삭제/이동, 멤버 합류 시 활동 로그가 자동 기록된다.
- 보드 우측 패널에 최근 50개 활동이 최신순으로 표시된다.
- 각 로그 항목에 행위자 이메일, 행위 내용, 상대 시각(N분 전)이 표시된다.

### FR-10: 실시간 동기화
- 팀원이 카드를 추가/이동/삭제하면 새로고침 없이 즉시 내 화면에 반영된다.
- 활동 로그도 실시간으로 업데이트된다.

## 5. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- 페이지 로드 및 카드 목록 조회 2초 이내

### NFR-02: 호환성
- 최신 Chrome, Firefox, Edge, Safari 지원
- 모바일 터치 이벤트는 v4.0 로드맵

### NFR-03: 보안
- Supabase Row Level Security(RLS)로 보드 멤버십 기반 데이터 격리
- anon key는 클라이언트에 노출되지만 RLS로 보호됨

### NFR-04: 유지보수성
- HTML/CSS/JS/Auth/Invite를 각각 별도 파일로 분리
